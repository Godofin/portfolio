Sorveteria Skimo - Criação DataWareHouse

Objetivos

O objetivo principal deste projeto é aplicar, de forma prática, os conhecimentos adquiridos durante os estudos de Business Intelligence. Mais especificamente, almejamos utilizar essas habilidades para coletar, processar e analisar dados relevantes para a tomada de decisões estratégicas em um determinado contexto empresarial. Para isso, planejamos utilizar ferramentas e técnicas de BI, como a criação de dashboards e relatórios personalizados, a fim de identificar tendências e padrões de comportamento que possam auxiliar na otimização de processos e no aumento da eficiência dos negócios.

Ferramentas Utilizadas no projeto

  • Excel: Base dos dados;
  • Integration Services: Ferramenta de ETL(Extract Transform Load);
  • Power BI: Realização da análise dos dados;
  • Figma: Desing do Dashboard;
  • DbDiagram.io: Lugar para realização do diagrama do banco de dados;

Fonte de dados

  • A fonte de dados utilizada para esse projeto, foi disponibilizada pela empresa Alura, que é uma escola de educação no rumo da tecnologia auxiliando com os conhecimentos.
  • A fonte de dados, foi disponibilizada em formato de arquivo .csv, que é um arquivo de texto, que contém dados separados por vírgulas, podendo ser aberto em qualquer editor de texto.
  • A fonte de dados, contém informações de vendas de uma sorveteria, que foi disponibilizada em 6 arquivos.
  • Vendas - Registro de todas as transações comerciais realizadas pela empresa.
  • Produtos - Catálogo de produtos oferecidos pela empresa.
  • Clientes - Informações dos clientes cadastrados, incluindo dados pessoais e de contato.
  • Pedidos - Registros de todos os pedidos realizados pelos clientes.
  • Itens Pedidos - Detalhes dos produtos incluídos em cada pedido.
  • Categoria - Classificação dos produtos em categorias específicas para facilitar a organização e análise dos dados.

Modelagem do Data Warehouse

Para iniciar o projeto, foi necessário fazer a modelagem do Data Warehouse, para isso, foi utilizado o Figma, que é uma ferramenta de prototipação de interface, que permite criar wireframes, protótipos de baixa e alta fidelidade, além de permitir a colaboração em tempo real. E também, bastante utilizada em times de scrum para realização de retrospectivas.
Dentro do figma, foi utilizado o modelo de dados floco de neve, que é um modelo de dados que permite a integração de dados de diferentes fontes, e também, permite a integração de dados de diferentes níveis de granularidade.

ETL - Extract Transform Load

No Integration Services, ETL significa Extrair, Transformar e Carregar. Essa é uma abordagem comum para integração de dados, que envolve extrair dados de diversas fontes, aplicar transformações para limpar, combinar ou enriquecer os dados e, em seguida, carregar os dados resultantes em um destino, como um banco de dados ou um data warehouse.
O processo ETL é uma parte fundamental do SSIS (SQL Server Integration Services), que é uma plataforma de integração de dados desenvolvida pela Microsoft. No SSIS, o processo ETL é executado em pacotes, que são fluxos de trabalho que definem as etapas de extração, transformação e carregamento dos dados.

Na etapa de extração, os dados são lidos de várias fontes de dados, como arquivos, bancos de dados, aplicativos ou serviços web. Na etapa de transformação, os dados são limpos, validados, enriquecidos ou manipulados de outras maneiras, conforme necessário, para atender aos requisitos de negócios. Por fim, na etapa de carregamento, os dados são carregados em um destino, como um banco de dados ou um data warehouse.
O SSIS oferece muitos recursos e ferramentas para a realização de ETL, como conectores de dados pré-definidos, transformações personalizadas, gerenciamento de metadados e gerenciamento de erros. O objetivo final do processo ETL é garantir que os dados estejam prontos para análise e uso pelos usuários finais.
